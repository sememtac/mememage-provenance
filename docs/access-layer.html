The Lock

how a chain stays sovereign

The Premise

Every Mememage record has three pieces of information about itself: who made it (the creator's signing key), what it is (the content hash), and who can read it (an optional password set at conception).

A signature proves authorship. A hash proves integrity. A password controls access. The first two are mandatory. The third is optional — if you want a chain to be public, set no password and everything stays public forever. If you want some part of your work to be private, set a password at the genesis mint and the entire chain becomes sealed under it.

We are the lock. You bring your key.

The Trinity

Three pillars hold the system. Each does one thing. Together they make a record both verifiable and sovereign.

⚷
Key
Identity
Ed25519 signing key. Proves you made this. Lives in ~/.mememage/private.key. Lose it = identity loss. Compromise = revoke and re-key.
#
Hash
Integrity
SHA-256 over canonical record fields. Proves what the record contains. Baked into the bar pixels and the watermark. Tamper detection without a server.
⌬
Password
Access
Optional. Set once at genesis. AES-256-GCM encrypts the protected fields. Whatever you type becomes the key to the chain. We never see it.

Three Chain Visibilities

The contract is set at the genesis mint — the first conception in a constellation. After that, every child inherits the same lock setting. You cannot retroactively gate an open chain. You cannot leave a sibling unlocked when the rest are sealed. The mode is recorded in the chain_visibility field of every record.

◯ Public Chain
No password ever set. Every record is fully visible — prompt, celestial reading, machine vitals, rarity, thumbnail. The bar works, the cert renders, anyone can read everything. GPS still rides the time-lock puzzle (anyone in ~10 years).
chain_visibility absent
◐ Light Energy
Password set at genesis, used only to seal the GPS coordinates. The cert renders fully without unlock — prompt, birth, rarity, thumbnail are all plaintext. Only the GPS field is gated: time-lock for strangers, password for the creator.
chain_visibility: "light_energy"
+ gps_password_locked
● Dark Matter, Locked
Password set at genesis, the entire soul is sealed. Decoder shows identity + signature badges + chain navigation. Every protected field reads "[encrypted]". The full plaintext lives in a single encrypted_fields blob — wrong password, no access.
chain_visibility: "dark_matter"
+ encrypted_fields + gps_password_locked
◑ Dark Matter, Unlocked
Same record after the viewer enters the right password. The browser decrypts encrypted_fields via SubtleCrypto, substitutes plaintext back into the protected fields, recomputes the content hash to verify. WITNESSED becomes provable. Refresh = sealed again.

WITNESSED is provable on public + light energy records without password. On dark matter records it's pending until unlock — once the soul decrypts, the decoder substitutes plaintext back in and recomputes the content hash to verify integrity.

The envelope, byte by byte

salt   16 bytes hex  — fresh per record, feeds PBKDF2 (rainbow-table defense)
iv     12 bytes hex  — AES-GCM nonce, fresh per encryption
ct     N  bytes hex  — AES-256-GCM ciphertext (no tag appended)
tag    16 bytes hex  — GCM authentication tag, separate field

Wrong password → GCM tag fails → InvalidTag exception → Wrong password in the UI. No partial decryption, no oracle, no timing leak.

The UX, End to End

1

At Genesis Mint

On the GPS-capture page, the creator chooses chain_visibility: public, light energy (GPS-only seal), or dark matter (whole soul sealed). For light or dark, a password input appears. Whatever they type becomes the AES key for the entire chain. Genesis is binding — every subsequent conception inherits the same setting.

2

Server Encryption

mememage/access.py derives a 256-bit key from the password (PBKDF2-SHA256, 600,000 iterations, fresh per-record salt). Light energy: encrypt only GPS coordinates, append gps_password_locked. Dark matter: collect all PROTECTED_FIELDS into one JSON blob, encrypt as encrypted_fields, replace the source fields with the "[encrypted]" sentinel. The server never stores the password.

3

Public Distribution

The record uploads to Internet Archive (or any host). Anyone can fetch it. Public + light energy: visible on inspection. Dark matter: only the identifier, signature, and chain navigation are readable; the rest is opaque ciphertext.

4

Decoder, First Render

The decoder pulls the soul, verifies the signature, checks chain_visibility. Public chain: paint the full cert. Light energy: paint the full cert with a password input in the GPS section. Dark matter: paint identity + chain navigation, replace each protected section with a sealed card, surface a top-of-cert password input.

5

Unlock

Viewer types the password, presses unlock. The browser runs PBKDF2 + AES-GCM via SubtleCrypto. Light energy: decrypts gps_password_locked, swaps the GPS row to live coords. Dark matter: decrypts encrypted_fields, parses the JSON blob, substitutes plaintext back into every protected field, recomputes the content hash to verify integrity, and re-renders the entire cert. Wrong password = GCM auth fails = Wrong password, no leak.

6

Chain Traversal

Click a parent or sibling. The decoder fetches the next record, sees the same chain's chain_visibility and envelopes. If the password is still in memory (session only), it auto-decrypts. Refresh the page = forgotten. localStorage never holds the key.

Threat Model

Two secrets, two threat levels, two recovery paths. The password is casual. The key is sacred.

What the password protects

  • Light energy: Strangers locating you via GPS before the 10-year time-lock matures
  • Dark matter: Strangers reading your prompts, seeing your faces, knowing what your machine looked like at conception, or computing the rarity of your output
  • Mass-scraped public archives gaining intelligence on your craft
  • Algorithmic correlation across your output (a leaked thumbnail dHash can match across thousands of records — dark matter blocks that)

What the password can't protect

  • Anyone who has the password — share it with one person, share it with everyone
  • The fact that you made the record (signature is always public)
  • The record's existence and chain position (identity + parent_id + constellation always visible)
  • The image's pixels if you posted them anywhere — encryption protects the soul, not the body
  • You from forgetting your own password — there's no recovery, no backdoor

Different concerns from the signing key:

  • Password compromise = privacy leak. Someone who shouldn't have your prompts now has them. Mint a new chain with a new password — old chain stays decrypted by the leaked key.
  • Key compromise = identity theft. Someone can sign as you. Run mememage revoke (publishes pre-signed revocation cert), then mememage keygen --force + new genesis. The decoder honors revocation.

Multiple Chains

A creator can run several chains in parallel — different visibilities, different audiences, different keys to the same lock shape:

  • Chain A · public — your portfolio, indexable, citable. No password ever.
  • Chain B · light energy — public-feeling work where you'd rather strangers not be able to map your studio. GPS sealed, everything else visible.
  • Chain C · dark matter, password X — private commissions. The password goes to the patron and only the patron sees the soul.
  • Chain D · dark matter, password Y — your studies, kept entirely for yourself.

The system doesn't know about tiers, subscribers, or platforms. If you want to tie passwords to subscriber lists or token gates, that's your business on your surface. We hold up the shape of the lock, not the key management.

The Line in the Sand

This is not C2PA. There is no consortium, no certificate chain, no institutional trust, no subscription. The creator holds their own key. Any artist can sign their own work — pip install mememage and a key is all it takes. The system supports creators who choose to seal their work. It does not require it. It will never require an account, a platform, or permission from anyone.

The artist is the authority.
