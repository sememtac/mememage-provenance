Soul Field Audit — hash_version: 1

Every field a V1 mint writes, in the order it appears in a .soul file on disk. Scannable scalars at the top, structured dicts in the middle, opaque blobs at the bottom. 31 active fields across 10 sections; dropped fields documented at the end. The on-disk order is cosmetic — readers re-canonicalize (sort_keys=True) before hashing, so the layout is free to be reorganized for human readability without touching the content hash.

Keep necessary, stays in V1 Dropped removed pre-launch in hash tamper-evident unhashed display / post-mint

Identity & lineage

1.identifierstrKeepin hash

Soul-discovery key. Format: mememage-XXXXXXXXXXXX (12 hex chars). Genesis is the special value mememage-0000000000000000. Stamped into the bar.

Used by: bar payload • lineage walk (parent_id) • soul-discovery URL fragment

"identifier": "mememage-22dd171b5d64"
2.content_hashstr (16 hex)Keepcomputed, then stamped

SHA-256 of canonical JSON over V1 inclusion set, first 16 hex chars. The integrity authority. Baked into the bar pixels.

Used by: bar payload • WITNESSED check • Ed25519 sig payload

"content_hash": "8ec3f6dd1c47d4d7"
3.hash_versionintKeepin hash

Which inclusion set applies. Dispatch field for future bumps. V1 is the launch canon. In the hash so an attacker can't downgrade the version field to dispatch the verifier to a different (more permissive or future) inclusion set.

Used by: _inclusion_set_for() dispatch • verify.js walker

"hash_version": 1
4.parent_idstr | nullKeepin hash

Identifier of the previous mint on this chain. Forms a linked list back to genesis (parent_id: null).

Used by: lineage walk • chain reassembly • constellation grouping

"parent_id": "mememage-prev123abc"
5.creator_namestrKeepunhashed

Display name the signer claims. Arbitrary text, set at mememage keygen --name "…" time. TOFU pins the (fingerprint, name) pair.

Used by: cert "Signed by" • TOFU first-seen prompt

"creator_name": "Andy"

Constellation & decoder

11.constellation_hashstr (16 hex)Keepin hash

SHA-256 of the canonical celestial subset of birth (sun, moon, moon_phase, mercury…saturn, angular_spread), first 16 hex. Drives constellation naming + visual pattern. All siblings in a constellation share the heart star's value.

"constellation_hash": "86026f0bcbe1a146"
12.constellation_namestrKeepin hash

Derived from constellation_hash + Age (via Sumerian phonetic table in constellation.py). Persisted so the family-claim label travels with the record without needing the lookup table or the Age at read time.

"constellation_name": "Ribimul"
13.constellation_indexint (0-11)Keepin hash

0-indexed birth order within the constellation. Heart star = 0; next 11 siblings = 1..11. Stored as int so canonical JSON stays ASCII-safe (Greek letter would serialize as \u03b1 escape and risk mismatched normalization between Python and JS hashers). Display layer maps to Bayer letter via 12-element lookup: 0→α, 1→β, …, 11→μ.

Used by: cert "Star" display • validator audit ("ε (4)") • webhook templates ({{constellation_star}} rendered as Greek at the surface via _bayer_letter())

"constellation_index": 4   // renders as 'ε' on every display surface
14.heart_star_idstrKeepin hash

Identifier of the heart star (first record in the constellation cycle). Every sibling carries the same value.

"heart_star_id": "mememage-22dd171b5d64"
15.decoder_hashstr (16 hex)Keepin hash

SHA-256 of the complete assembled decoder HTML at the moment of seal. Constant for the entire Age. Chunk-injection defense: forged chunk → wrong reassembly → caught.

"decoder_hash": "1e6fc76c65c34000"

Fingerprints, traits & rarity

16.machine_fingerprintstr (8 hex)Keepin hash

First 8 hex of SHA-256(cache=X|cores=Y|cpu=Z|gpu=N|ram=R) over the stable fields of birth.machine. Stable per machine; same hardware → same fingerprint every conception.

Used by: validator Fingerprint display • cert machine-band visual seed • personality tracker file (local)

Why persisted: derivable from birth.machine but kept anyway — dropping would force async SubtleCrypto in initMachineBand + 2 display sites, and the ~25-byte saving doesn't pay for itself. The persisted form is also tamper-evident (in hash) without requiring readers to re-derive.

"machine_fingerprint": "e3b0c442"
17.key_fingerprintstr (formatted)Keepin hash

First 16 hex chars of SHA-256(public_key bytes), in 4-char groups separated by colons. Stable, human-friendly identifier of the signer. In the hash alongside public_key for the signer-swap defense.

Used by: TOFU localStorage key (8+ sites in ui.js) • IA keychain collection name (mememage-keychain-<fp>) • cert display • validator audit • webhook templates • CLI verify

Why persisted: derivable from public_key but kept anyway — dropping would force async SubtleCrypto on every TOFU check / keychain discovery / cert render. ~25-byte saving doesn't pay for itself.

"key_fingerprint": "86cb:4ed6:af3f:d6c5"

Cryptographic proof

25.signaturestr (128 hex)Keepunhashed

Ed25519 signature over identifier + \0 + content_hash + \0 + sha256(thumbnail). The thumbnail hash extends the payload so a thumbnail swap also breaks AUTHENTICATED (closing the gap where only EMBODIED — and only when the user drops the image — would catch it).

Used by: AUTHENTICATED check (verify.js)

Pipeline: deferred to post-mint — the signature rides into IA on the same _patch_record reblast that already delivers the thumbnail. Initial mint upload contains no signature (~1s window before the patch lands). Optional (records without a signing key remain unsigned).

"signature": "a3f9b2c8d1e4f602…d8e1"  // 64-byte Ed25519, hex-encoded
// Payload bytes: identifier + \0 + content_hash + \0 + sha256(thumbnail_string)
// thumbnail empty / absent  →  payload ends with trailing \0 + ""
26.public_keystr (64 hex)Keepin hash

Ed25519 public key bytes, hex-encoded. Verifier needs it to check the signature. In the hash — closes the signer-swap attack: without this binding, an attacker could strip signature + public_key, drop in their own key, sign the existing identifier + content_hash, and TOFU would seat them as the creator for any first-touch viewer. With public_key in the hash, swapping the key changes the hash, which changes what the bar must contain, which breaks the visual proof. Signer becomes part of the artifact's identity, not bolted on.

Used by: signature verification • TOFU consistency check

"public_key": "86cb4ed6af3fd6c51e9e346b23942f90d5b15e0f3bf3760dfc033313553150ad"

Payload integrity

27.chunks_rootstr (16 hex)Keepin hash

SHA-256 over canonical JSON of {layer_name|pinned_role: chunk.hash}, first 16 hex. Aggregates the per-chunk hashes (already present inside each chunk) into a single tamper-evident root that lives in the content hash. Any chunk swap or chunk-hash tamper breaks WITNESSED — without forcing the verifier to download the bulk chunk data. Lightweight By Word verification still works against just the soul JSON. Absent on pre-seal chains (no chunks → no root, same pattern as gps_time_locked on gps_source: "none" chains).

Used by: WITNESSED check on records that carry chunks • validator chunk-integrity panel

"chunks_root": "a3f9b2c8d1e4f602"
// = sha256({"decoder":"c30082d5fccb","truth":"b03700e874c8"}, canonical)[:16]

Post-mint payload (opaque blobs)

Everything added after content_hash was computed. Not in the hash inclusion set (the hash exists before these bytes are produced). Chunk bulk data isn't hashed directly — chunks_root (#26) seals the chunk hashes into the content hash, so swapping chunk data after the fact is caught without forcing chunk download. Thumbnail integrity rides on the Ed25519 signature.

28.thumbnailstr (data URI ~1KB)Keepunhashed (but signed)

256×256 JPEG q78 of the post-mint image (bar + watermark embedded). Used for EMBODIED dHash portrait check. Generated AFTER bar/watermark so the comparison is apples-to-apples with the dropped image. Not in content_hash (chicken-and-egg: bar carries the hash, so re-embedding to bind the thumbnail would create an infinite loop). Bound by signature instead — the Ed25519 payload includes sha256(thumbnail), so any post-mint thumbnail swap breaks AUTHENTICATED (not just EMBODIED).

Used by: EMBODIED check (verify.js's computeDHash) • cert portrait display • signature payload

"thumbnail": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAAAQABAAD…"

Dropped pre-launch (with rationale)

Fields removed during the V1 audit. Decision log entries live in docs/plans/soul-field-audit.md.

rarity_score Dropped

Pure sum of rarity dice + machine_signature + sigil.points (clamped 0-255). Derivable from the rarity dict; reconstructed by RarityScore.fromRecord() in docs/js/rarity-helpers.js. Webhook surface still computes it server-side.

distribution Dropped

Souls are surface-agnostic — the artifact shouldn't carry a list of every mirror it landed on. The full publish-results map stays server-side (returned by channels.blast(); surfaced via webhook {{distribution}} + dashboard handoff). Mirror discovery is operational, not artifact-level.

url Dropped

Pre-launch, "legacy bar consumer" / "Discord notification" rationale was hypothetical — zero JS readers touched meta.url. MintResult.url still bubbles the primary URL back to callers, so server-side surfaces (webhook templates, dashboard handoff) keep working without the URL landing in the artifact.

birth_readings / birth_temperament / birth_summary Dropped

All three derived from birth_traits (integer codes) via fixed lookup tables. Reconstructed at display via docs/js/birth-text.js (JS) and mememage/temperament.py (Python). The codes are the canonical signal; the readable strings were redundant.

mode Dropped

Always "text-to-image" in practice. Resurrect when img2img / inpaint pipelines actually start using it; meanwhile a single-value field doesn't earn its slot.

cfg (renamed → cfg_scale, inside origin)

Three-letter abbreviation was opaque. Now lives inside origin with the full name. Generation pipelines accept either name at the input boundary; only the canonical V1 name persists.

unet (renamed → model, inside origin)

"UNet" leaked an SD-pipeline implementation detail. Renamed to model (universal: works for FLUX, Midjourney, anything else). Now inside origin.

constellation_star (renamed → constellation_index, int)

Greek letter ("ε") became "\u03b5" in canonical JSON — exactly where encoding bugs hide between Python's ensure_ascii=True and JS's matching escape pass. Storing the position as int is ASCII-safe and matches what the field actually is. Display layers reconstruct the Bayer letter.

_about (renamed → about)

Leading underscore was the only field of its kind in the schema — JS-style "private/meta hint" that didn't belong. Rest of the schema uses snake_case as a word separator, not a privacy marker.

born (renamed → birth)

"born" is a past-participle adjective; "birth" is the proper noun for the moment-of-birth state. Reads better as a JSON key.

birth.gps_locked / gps_encrypted (renamed + relocated)

GPS pulled out of birth entirely — the time-lock puzzle is now gps_time_locked at top-level; the password envelope is gps_password_locked. Chains with gps_source: "none" produce a symmetric record (no GPS keys at all instead of half-empty birth).

chunks.decoder.{age,age_name,reassembly} / chunks.proof.{cycle_length,day,seal} Dropped

Demo-flavor extras that special-cased two layer names. Now every layer emits the same five-field shape regardless of name. Age info hoisted to top-level age int (display name via 12-element AGE_NAMES lookup in docs/js/age-names.js).
